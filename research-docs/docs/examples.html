What is Internet Server Platform (ISP)

ISP - is a software platform which having the core only it does nothing useful, but in combination with modules
that provide custom functionalities becomes a shapeless and powerfull tool that has as aim to channel data between people.
The core is nothing, a module is nothing, together they become an extension to each other.

The things that ISP provides in a transparent manner for a module are:
    o Stream oriented connections. So far TCP is the only one used.
    o Secured stream oriented connections. So far SSL over TCP is the only one used.
    o Secured private spaces called 'bottles'. A module can have any number of bottles.
       For making things uber secure bottles use two keys.
    o Data gatewaying in format n<->m.
    o Flexible techniques of securing against potential intruders.

The things that ISP provides for the person that configures it:
    o trivially simple configuration, here is an example.

What a module is likely to do if one is imminent to explode into life:
    o inherit a basic interface provided by the ISP so they can communicate
    o implement the only one method to handle the request arrived to this module
    o use ISP's facilities to transfer data in the direction it wants
    o if the module wants a secure connection, then it asks for it and it will be provided.
    o at the moment the request arrives, if the module was asking for a secure connection,
       the data that came was comming on secure connection, otherwise it won't come.
